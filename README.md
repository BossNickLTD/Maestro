# Maestro — AI-приёмщик заказов для ресторанов

## Суть

Голосовой терминал на каждом столе, который заменяет официанта как «буфер между гостем и кухней». Убирает ошибки передачи, языковой барьер, задержки — и снижает расходы на персонал.

## Как это работает

```
Гость нажимает кнопку на столе
       ↓
Говорит: «цезарь с курицей без помидоров и большой латте»
       ↓
Whisper (OpenAI) → транскрибация речи в текст (~$0.006)
       ↓
LLM (GPT-4o-mini / LLaMA) → извлечение структурированного заказа (~$0.002)
       ↓
Система проверяет:
  - блюдо есть в меню?
  - блюдо не в стоп-листе?
  - модификатор подходит к блюду?
       ↓
Голосовое подтверждение: «Цезарь с курицей без помидоров, латте большой. Я всё верно понял?»
       ↓
Заказ уходит на кухонный принтер и в кассу
```

## Роль официанта меняется

Официант становится **хостайером**:
- Встречает, усаживает, открывает сессию за столом
- Подаёт еду, проверяет что всё нравится
- Продаёт десерты и алкоголь вживую — через харизму, не через меню
- Если гость просит человека — подходит (но 80% заказов проходят через голосовой ввод)

## Проблемы, которые решает

| Проблема | До | После |
|---|---|---|
| Ошибки стоп-листа | Официант забыл спросить | Система проверяет автоматически |
| Ошибки передачи | «Вам цезарь с креветками?» — «Нет, с курицей» | Заказ структурирован, подтверждён голосом |
| Время от заказа до подачи | Официант отошёл к другому столу | Заказ уходит мгновенно |
| Стоимость персонала | $1000/мес за квалифицированного официанта | $600/мес за хостайера |
| Языковой барьер | Официант не знает английский | AI говорит на любом языке |

## Стратегия (5 слоёв)

### Слой 1 — Устройство (zero margin, захват)
Терминал на каждом столе. Себестоимость ~$40. Отдаётся почти даром или бесплатно за контракт на 12 месяцев. Цель — физическое присутствие в каждом ресторане.

### Слой 2 — SaaS (маржа 80%)
- $200/мес за обработку заказов
- $100/мес за аналитику (какие блюда перезаказывают, возвращают, модифицируют)
- $100/мес за AI-апселлинг (терминал сам предлагает десерт/кофе в конце трапезы)

### Слой 3 — Данные (настоящие деньги)
- Каждое произнесённое слово от гостя → транскрибация → структурированные данные
- Спекулятивная data-лента поставщикам: «через 2 недели пик спроса на лосось, завези на 30% больше»
- Аналитика для сетей: «ваша концепция не заходит в этом районе, меню надо менять»

### Слой 4 — Рынок (монополия)
- 1000 ресторанов на терминалах → маркетплейс ингредиентов
- Рестораны заказывают продукты через Maestro, потому что AI предсказал спрос
- Maestro становится дистрибьютором еды

### Слой 5 — Финансы (экзит)
- Контроль потока orders + payments + inventory
- Stripe для физической еды. Каждая транзакция проходит через Maestro
- Exit: $2B+ покупка Toast, Uber Eats или Deliveroo

## Экономика для ресторана (40 мест, средний чек $15, оборот $100k/мес)

| Статья | До | После |
|---|---|---|
| Персонал | 3 официанта × $1000 = $3000 | 2 хостайера × $600 = $1200 |
| Ошибки/списания | 2% = $2000 | 0.5% = $500 |
| AI-обработка | — | $150/мес |
| **Итого** | **$5000** | **$1850** |

Экономия: ~$3150/мес + повышенный оборот за счёт туристов (языковой барьер снят).

## Почему это сработает

1. **Не замена людей, а снятие узкого места.** Рестораторы и гости ненавидят «AI вместо людей». Мы продаём «меньше ошибок и быстрее обслуживание».
2. **Zero интеграции.** Работает через готовые API. Касса — через простой REST или Telegram-бот.
3. **Ретрофит.** Можно начать как QR → веб-приложение (гость через телефон). После подтверждения — поставить дешёвые спикерфоны.

## Единственное «но»
Ресторатор должен хотеть изменить процесс, а не просто «купить AI». Продажа — не софт, а реинжиниринг кухни через устранение ошибок передачи.

## Дорожка

| Этап | Что делаем | Результат |
|---|---|---|
| **1. Бэкенд** | FastAPI + эндпоинт /order + Groq | Принимает текст → JSON-заказ |
| **2. Меню** | JSON-файл меню, стоп-лист, валидация | Ни одно «драконье яйцо» не пройдёт |
| **3. Нормализация** | Синонимы: цезарь = салат цезарь = цезарь с курицей | Одно блюдо — один ID |
| **4. Принтер** | ESC/POS вывод на термопринтер | Чек печатается |
| **5. Dashboard** | Веб-страница с заказами и стоп-листом | Ресторатор управляет |
| **6. Звук** | STT (Whisper) + TTS | Голосовой ввод-вывод |
| **7. Железо** | Raspberry Pi + кнопка + микрофон | Физический терминал |
| **8. Деплой** | VPS + домен + HTTPS | Работает из любого кафе |
