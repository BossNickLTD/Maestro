# Maestro — прогресс проекта

## Дорожка

| Этап | Что делаем | Статус |
|---|---|---|
| **1. Бэкенд** | FastAPI + эндпоинт /order + Groq | ✅ |
| **2. Меню** | JSON-файл меню, стоп-лист, валидация | ✅ |
| **3. Нормализация** | Синонимы, fuzzy-match, модификаторы, промпт с меню | ✅ |
| **4. Принтер** | ESC/POS вывод на термопринтер | ✅ |
| **5. Dashboard** | Веб-страница с заказами и стоп-листом | ✅ |
| **6. Звук** | STT (Whisper) + TTS | ✅ |
| **7. Железо** | Raspberry Pi + кнопка + микрофон | ⬜ (опционально) |
| **8. Деплой** | VPS + домен + HTTPS | ⬜ |

## Этап 1 — Бэкенд ✅

**Файлы:** `main.py`

**Что сделано:**
- FastAPI сервер с эндпоинтом `POST /order`
- Подключение к Groq API (LLaMA 3.3 70B) через `.env`
- Swagger UI на `http://127.0.0.1:8000/docs`
- Очистка markdown-обёртки из ответа LLM
- Обработка `unknown` — что LLM не смогла сопоставить с меню
- Фильтр: слова, входящие в названия найденных блюд, не попадают в `unknown`

**Коммит:** `afbba0b`

## Этап 2 — Меню ✅

**Файлы:** `menu.json`, `menu_utils.py`

**Что сделано:**
- Структура меню: items, aliases, modifiers, stop_list
- 9 позиций в меню (4 категории)
- Валидация заказа по меню: проверка существования блюда
- Проверка стоп-листа
- Синонимы (aliases): «цезарь» → «Цезарь с курицей», «чай» → «Чай чёрный» и т.д.
- Подсчёт общей суммы заказа

**Коммит:** `1700c9b`

## Этап 3 — Нормализация ✅

**Файлы:** `menu.json`, `menu_utils.py`, `main.py`

**Что сделано:**
- Промпт LLM теперь включает полное меню с ценами и модификаторами
- LLM возвращает названия строго из меню
- Три слоя поиска блюда:
  1. Точное совпадение по названию
  2. Совпадение по алиасам (синонимы)
  3. Нечёткий поиск (SequenceMatcher, порог > 0.6)
  4. Пересечение по токенам (словам) с порогом > 40%
- Извлечение модификаторов: молоко, сахар, добавки
- Фильтрация `unknown`: слова из названий найденных блюд и модификаторы исключаются

**Тестовые кейсы (все пройдены):**

| Ввод | Результат |
|---|---|
| «два латте и круассан» | 2 позиции, 320 руб, 0 ошибок |
| «борщ и лате» | 1 позиция (латте), 120 руб, 1 ошибка (борщ) |
| «салат цезарь с курицей и чай с лимоном без сахара» | 2 позиции, 0 ошибок, лимон как модификатор |
| «карбонара и большой латте на соевом» | 2 позиции, 0 ошибок (большой — не unknown) |
| «два цезаря с креветками, пасту и чай зелёный без сахара» | 3 позиции, 0 ошибок, 1270 руб |

**Модификаторы в меню:**
- Латте / Капучино: молоко (обычное, соевое, миндальное, без лактозы)
- Чай чёрный / зелёный: сахар (без сахара, 1/2 ложки), добавки (лимон, мёд)

**Коммит:** `d4fcb83`

## Этап 4 — Принтер ✅

**Файлы:** `printer.py`

**Что сделано:**
- Модуль печати `print_order()` — генерирует чек с шапкой, позициями, модификаторами, итогом
- Формат чека: название заведения, номер заказа, стол, дата, список позиций с quantity × price, итого
- Сохранение в файл в папку `receipts/` (режим разработки)
- Счётчик заказов с сохранением между перезапусками (`_counter.json`)
- Интеграция с эндпоинтом `POST /order` — каждый заказ автоматически печатается
- `order_num` возвращается в ответе API
- Поле `table` в запросе (опционально, для чека)

**Пример чека:**
```
========================================
            M A E S T R O
  заказ #3 | Стол 5
  17.07.2026 16:30:07
========================================
  Латте
    2 x 120 = 240
  Круассан
    1 x 80 = 80
----------------------------------------
  ИТОГО:                   320
========================================
        Спасибо за заказ!
========================================
```

**Для прода:** заменить запись в файл на отправку байтов ESC/POS через USB.

**Коммит:** `8664b32`

## Этап 5 — Dashboard ✅

**Файлы:** `templates/dashboard.html`

**Что сделано:**
- HTML-страница с Tailwind CSS (CDN, без сборки)
- Автообновление каждые 10 секунд
- Статистика: заказов сегодня, выручка, средний чек
- Список последних 20 заказов (номер стола, позиции, сумма, ошибки)
- Стоп-лист: кнопка «стоп»/«вернуть» для каждой позиции меню
- Эндпоинты:
  - `GET /dashboard` — страница
  - `GET /history` — история заказов (из `_orders.json`)
  - `GET /menu-data` — меню со стоп-листом
  - `POST /stop-list` — переключить позицию в стоп-листе
- История заказов сохраняется в `_orders.json` (каждый POST /order)
- Стоп-лист сохраняется в `menu.json`

**Коммит:** `d04d7a1`

## Этап 6 — Звук ✅

**Файлы:** `voice.py`

**Что сделано:**
- TTS (Text-to-Speech) через edge-tts (бесплатно, без API-ключа)
  - Голос: ru-RU-SvetlanaNeural
  - Функция `tts(text)` — сохраняет MP3 в папку `audio/`
- STT (Speech-to-Text) через Groq Whisper (`whisper-large-v3-turbo`)
  - Эндпоинт `POST /transcribe` — аудиофайл → текст
- Полный голосовой заказ: `POST /voice-order`
  - Аудио → транскрибация → LLM извлекает заказ → валидация по меню → печать чека → сохранение истории → TTS-ответ
  - Возвращает: текст, items, errors, total, order_num, audio_url
- `GET /tts?text=...` — любой текст → MP3
- Аудиофайлы раздаются как статика через `/audio/*`
- Папка `audio/` — создаётся автоматически

**Коммит:** `dfb6267`

## Этап 7 — Железо ⬜ (опционально для MVP)

> ⚠️ **Важный момент для прода:** этот этап можно пропустить при запуске.
> Первый MVP тестируется с телефона (браузер → микрофон → API).
> Железо докупается и внедряется за 2 дня после появления первого платящего клиента.

**Что потребуется для прода:**

| Компонент | Назначение | Цена |
|---|---|---|
| Raspberry Pi Zero 2W | Мини-компьютер для кода | ~$15 |
| USB-микрофон | Запись голоса гостя | ~$10 |
| Колонка / динамик | Воспроизведение TTS-ответа | ~$5 |
| Кнопка (GPIO) | Запуск записи | ~$1 |
| Корпус | Защита устройства | ~$5 |

**Как будет работать:**
1. Гость нажимает кнопку на терминале
2. Raspberry Pi записывает голос
3. Отправляет на сервер (`POST /voice-order`)
4. Получает ответ + аудиофайл
5. Проигрывает подтверждение через колонку

**Почему можно отложить:**
- На этапе MVP все эндпоинты (transcribe, voice-order, tts) уже готовы и работают
- Raspberry Pi — это просто тонкий клиент: запись → HTTP → воспроизведение
- Вся логика уже на сервере, железо ничего не решает

## Этап 8 — Деплой ⬜

**План:**
- Аренда VPS ($5-10/мес, DigitalOcean / Timeweb)
- Установка Python + зависимостей
- Запуск FastAPI через systemd (автозапуск при перезагрузке)
- Домен + HTTPS через Nginx + Let's Encrypt / Certbot
- Настройка `.env` с Groq API-ключом на сервере
- Открыть порт 80/443 в firewall

**Результат:** сервер доступен по https://maestro.domain.com из любого кафе.
